# 实验三成果目录

本目录对应《实验内容三.docx》中的要求：研讨传统软件开发过程模型与敏捷开发，阅读 Scrum 开发方法文档，完成可行性分析报告，并将工作文档和进度保存到协作开发平台。

## 文件清单

| 文件 | 说明 |
|---|---|
| `experiment_03_report.md` | 实验三主报告，包含过程模型比较、Scrum 理解、项目过程方案、风险管理和可行性分析报告定稿。 |
| `scrum_backlog.md` | 面向人才招聘系统的 Product Backlog、Sprint 划分和完成定义。 |
| `risk_register.md` | 项目风险登记册，包含风险分级、预防措施和应急预案。 |

## 审阅重点

1. 项目过程是否采用“适度前期设计 + Scrum 迭代 + 局部 XP 实践”的混合方式。
2. 风险分级和应急预案是否符合当前项目范围。
3. 可行性分析报告定稿是否可以作为课程提交版本。
