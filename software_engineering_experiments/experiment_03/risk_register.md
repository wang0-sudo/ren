# 人才招聘系统风险登记册

## 1. 风险评分规则

概率 P 和影响 I 均取 1 到 5 分：

| 分值 | 概率含义 | 影响含义 |
|---:|---|---|
| 1 | 很少发生 | 几乎不影响进度和质量 |
| 2 | 可能发生 | 有轻微影响 |
| 3 | 中等概率 | 影响部分功能或文档 |
| 4 | 较大概率 | 影响关键功能或进度 |
| 5 | 极大概率 | 可能导致项目无法按要求交付 |

风险值：

`R = P * I`

| 风险值 | 等级 | 管理策略 |
|---:|---|---|
| 1-4 | 低 | 记录并观察 |
| 5-9 | 中 | 制定预防措施 |
| 10-16 | 高 | 重点跟踪，准备应急预案 |
| 17-25 | 很高 | 优先处理，必要时调整项目范围 |

## 3. 高风险优先处理计划

| 风险 | 本周处理动作 | 期望结果 |
|---|---|---|
| R1 需求范围扩大 | 明确核心功能和扩展功能边界 | 后续实验不偏离主流程 |
| R2 进度估计不足 | 使用 Sprint Backlog 和项目跟踪表 | 每个实验有明确交付物 |
| R4 数据可靠性 | 记录当前文本存储风险 | 后续设计阶段考虑 SQLite 或备份策略 |
| R5 安全不足 | 在 SRS 和 SAD 中加入安全需求 | 后续实现密码哈希和最小权限设计 |
| R6 权限控制 | 在模型和测试中单列权限场景 | 降低越权操作风险 |
| R9 测试不足 | 后续建立回归测试清单 | 演示前有明确测试依据 |

## 4. 风险复盘机制

1. 每完成一个实验，更新风险状态。
2. 若风险值达到 10 以上，必须在报告中说明应对方案。
3. 若出现新风险，补充到登记册。
4. 若风险已经消除，可将状态改为 Closed，但不删除历史记录。
