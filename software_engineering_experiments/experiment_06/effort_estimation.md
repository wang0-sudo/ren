# 人才招聘系统工作量估算

## 1. 估算目标

本估算用于实验六“工作量估算”要求，目标是对人才招聘系统的开发规模、工作量和工期给出可解释的近似判断。估算不是商业合同报价，而是课程项目计划依据。

## 2. 规模依据

当前招聘系统核心代码主要位于：

| 文件 | 约行数 |
|---|---:|
| `qt_gui/recruitment_service.h` | 170 |
| `qt_gui/recruitment_service.cpp` | 819 |
| `qt_gui/login_window.h` | 23 |
| `qt_gui/login_window.cpp` | 305 |
| `qt_gui/dashboard_window.h` | 65 |
| `qt_gui/dashboard_window.cpp` | 1058 |
| `qt_gui/main.cpp` | 30 |
| 合计 | 2470 |

考虑空行、界面样板代码、构建脚本和后续文档/测试补充，课程项目估算规模取：

`2.5 KSLOC`

其中 KSLOC 表示千行源代码。

## 3. COCOMO II 简化估算

### 3.1 模型公式

采用 COCOMO II 后架构模型的简化形式：

`PM = A * Size^E * EAF`

其中：

| 参数 | 含义 | 本项目取值 |
|---|---|---:|
| PM | 人月工作量 | 待计算 |
| A | 校准常数 | 2.94 |
| Size | 代码规模，单位 KSLOC | 2.5 |
| E | 规模指数 | 1.08 |
| EAF | 工作量调整因子 | 0.90 |

### 3.2 参数理由

| 参数 | 理由 |
|---|---|
| Size = 2.5 | 当前招聘系统核心代码约 2.47 KSLOC，取整为 2.5。 |
| E = 1.08 | 项目规模小，需求较清楚，但仍有界面和文档变更风险，采用略高于线性的指数。 |
| EAF = 0.90 | Qt 和 C++ 有一定复杂度，但已有原型和本地运行环境，整体略低于普通新项目。 |

### 3.3 工作量计算

`PM = 2.94 * 2.5^1.08 * 0.90`

`2.5^1.08 ≈ 2.69`

`PM ≈ 2.94 * 2.69 * 0.90 ≈ 7.12 人月`

因此，人才招聘系统按 COCOMO II 简化估算的工作量约为：

`7.1 人月`

## 4. 课程项目修正

COCOMO II 更适合正式软件项目。课程项目存在以下差异：

1. 部分功能已经实现，不是从零开始。
2. 交付目标是课程演示和文档，不是生产系统上线。
3. 数据库、安全、部署、性能等要求相对轻量。
4. 大量 Qt 界面代码属于模板化和重复性实现。

因此，将模型估算折算为课程剩余工作量：

| 工作类别 | 估计工作量 |
|---|---:|
| 需求和 SRS | 0.8 人月 |
| 建模和架构设计 | 0.8 人月 |
| 已有核心功能整理和完善 | 1.2 人月 |
| 测试和缺陷修复 | 0.6 人月 |
| 文档、报告和答辩材料 | 0.8 人月 |
| 合计 | 4.2 人月 |

如果由 3 人小组完成，按每人每周约 0.25 人月投入：

`4.2 / (3 * 0.25) ≈ 5.6 周`

因此，项目在 6 周左右完成核心交付是可行的。

## 6. 估算结论

从 COCOMO II 简化模型看，人才招聘系统完整正式开发约需 7.1 人月；考虑已有原型、课程交付范围和轻量化要求，课程项目剩余工作量约为 4.2 人月。若按小组分工推进，6 周左右完成核心功能、SRS、SAD、测试和实验文档是可行的。
