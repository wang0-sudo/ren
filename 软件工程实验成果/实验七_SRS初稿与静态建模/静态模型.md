## 2. 领域类图

```mermaid
classDiagram
    class RecruitmentService {
        +loadOrSeed()

        +applyForPosition()

    class PersonalUser {
        int id
        string username
        string password
        string name
        string gender
        string phone
        string email

    }

    class Resume {
        string education
        string experience
        string skills

        int id
        string username
        string password
        string companyName
        string contactPerson
        string phone

        string introduction

    }

    class Position {
        int id
        int enterpriseId
        string title
        string location
        string salary
        string requirement
        string description
        bool active
    }

    class Message {
        int id
        string senderType
        string senderName
        string content
        bool handled
        string reply
    }

    class AuditStatus {
        Pending
        Approved
        Rejected
    }

    RecruitmentService --> PersonalUser

    RecruitmentService --> Position
    RecruitmentService --> Message
    PersonalUser *-- Resume
    PersonalUser --> AuditStatus
    EnterpriseUser --> AuditStatus

    PersonalUser "*" --> "*" Position : applies
```

## 5. 静态模型说明

1. 用例模型展示三类用户与系统功能的关系。
2. 领域类图直接对应当前代码中的核心结构体和服务类。
3. E-R 模型为后续从文本文件迁移到 SQLite 提供基础。
4. 模块结构图体现界面层、业务层和存储层的分层关系。
