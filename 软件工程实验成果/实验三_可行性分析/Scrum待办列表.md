# 人才招聘系统 Scrum Backlog

## 1. 产品目标

在课程项目周期内完成一个可运行、可演示、文档较完整的桌面端人才招聘系统。系统应覆盖个人用户、企业用户和管理员三类角色，形成“注册审核、岗位发布、岗位查询、岗位申请、留言处理”的基本业务闭环。

## 4. Definition of Done

一个功能或实验成果只有满足以下条件，才视为完成：

1. 功能可以在当前项目环境中运行或文档可以直接审阅。
2. 关键输入错误有提示或在风险中说明。
3. 对应 Markdown 文档已放入正确实验目录。
4. 已通过审阅。
5. 审阅通过后已提交并推送到 GitHub。

## 5. 迭代跟踪方式

建议在 GitHub Projects 中建立以下列：

| 列 | 含义 |
|---|---|
| Todo | 尚未开始的需求、文档或缺陷 |
| In Progress | 正在处理 |
| Review | 已完成，等待审阅 |
| Done | 已确认并推送 |

建议字段：

| 字段 | 示例 |
|---|---|
| Priority | High / Medium / Low |
| Type | Code / Document / Test / Research |
| Sprint | Sprint 0 / Sprint 1 / Sprint 2 / Sprint 3 / Sprint 4 |
| Estimate | 1 / 2 / 3 / 5 |
